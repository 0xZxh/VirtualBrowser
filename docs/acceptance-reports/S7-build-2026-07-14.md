## 追加备注 — 2026-07-14（adm-zip 缺失启动失败）

- **现象：** 安装 `VirtualBrowser-Setup.exe` 后启动 Uncaught Exception：`Cannot find module 'adm-zip'`（`server/lib/profile-sync.js` ← `native-runtime.js` ← desktop-shell）。
- **根因：** `build-client.ps1` 仅复制 `server/lib`，未复制 Node 解析所需的 `server/node_modules`；`adm-zip` 已在 `server/package.json` 声明，但安装包内缺失。
- **扫描：** `server/lib` 第三方 require 仅 `adm-zip`（`profile-sync.js` / `crx-store.js`）；`adm-zip` 无额外 npm 依赖。
- **修复：** staging 组装时复制 `server/node_modules/adm-zip`，并在构建中 `require` staging 内 `native-runtime` 做校验；smoke 增加从 `appRoot/server` 解析 `adm-zip`。
- **重构建：** `build-client.ps1 -CloudApiBase "http://127.0.0.1:3001" -SkipUiBuild` → Setup 大小约 486478547 bytes（22:03）；校验日志 `ok: staging resolve native-runtime + adm-zip`。
- **注：** 若旧 `packaging/staging` 被文件锁占用，脚本会改用 `staging-YYYYMMDDHHMMSS` 并传给 makensis（本机曾遇 `Chrome-bin\resources\app.asar` 锁定）。
