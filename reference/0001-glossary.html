VirtualBrowser 二开术语速查

后续课程与代码讨论统一使用下列定义。

架构层

server/
Vue 2 + Element UI 管理控制台，打包后嵌入 VirtualBrowser 安装包作为 WebUI。
worker/
Worker 相关 Vue 子项目，与独立浏览器窗口/Worker 实例 UI 相关。
native 桥
server/src/api/native.js 通过 Chromium WebUI 的 chrome.send(name, args) 调用 C++ 侧能力（如 getBrowserList、setBrowserList）。
浏览器环境 / 资产
一条指纹配置 + 独立用户数据目录，对应列表中的一行；CRUD 经 native 桥，非普通 REST。
automation/
Playwright 示例：用 launchPersistentContext + --worker-id 驱动已安装 VirtualBrowser。

权限与私有化

mock 登录
当前 server/src/api/user.js 仍请求 /vue-element-admin/user/*，私有化需换真实认证 API。
路由权限
permission.js 守卫 + asyncRoutes 按 meta.roles 过滤；页面/按钮级用 v-permission。
三层权限（二开目标）
① UI 可见性（路由/按钮）② 业务 API（谁能操作哪条环境）③ native 层（可选，最终数据落盘与启动）。

关键 chrome.send 方法（开源可见）

  • getBrowserList / setBrowserList — 环境列表
  • deleteBrowser — 删除环境
  • getRuningBrowser — 运行中实例
  • getGlobalData / setGlobalData — 全局配置
  • getBrowserVersion — 版本信息
