术语速查 Mission

Lesson 0001 · 约 15 分钟

架构地图与私有化路线图

私有化不是「换皮」这么简单：你要同时掌控 谁 能登录、谁 能操作哪些浏览器环境、以及 UI 如何调用底层指纹浏览器。

1. 你的 Mission（已写入仓库）

目标：私有化部署、定制 UI、账号 + 资产（浏览器环境）+ 权限体系，并为将来在通信层加业务逻辑留接口。 详见项目根目录 MISSION.md。

2. 开源仓库里有什么

要点：浏览器环境的增删改查不走你未来的业务 REST，而是 native.js → chrome.send。账号权限体系是你新增的后端 + 改 server 前端； 环境与指纹数据仍在 native / 本地存储，二者要在设计上对齐。

3. 私有化二开要动的三层

层 A — 表现层（最先动手） Logo、主题色、文案、菜单结构；改 server/src/layout、lang/、styles/。
层 B — 账号与权限（你的核心 Mission） 替换 api/user.js mock；登录拿 token + roles；permission.js 动态路由； 环境列表按租户/用户过滤（需业务 API 或扩展 native 元数据）。
层 C — 通信与业务钩子（未来） 在 chrome.send 前后加审计/配额；automation/ 接调度系统； 或通过 setGlobalData 下发业务配置给 native。

4. 推荐学习路线（后续课程）

  1. 0002 — 跑通 server 本地 dev，对照安装包里的页面
  2. 0003 — 完成第一个 UI 私有化改动（主题/Logo）
  3. 0004 — 读懂 permission.js + mock 登录，画权限数据流
  4. 0005 — 设计「用户 — 角色 — 浏览器环境」模型 MVP
  5. 0006 — native.js 与环境生命周期（启动/删除/导入导出）
